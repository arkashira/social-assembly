## 1. Acceptance Criteria
1. **Moderator role assignment on forum creation** – When a community moderator creates a new forum, the creator’s user record must have the `moderator` role for that forum stored in the database.  
2. **Persisted configuration** – Forum `name`, `description`, and `privacy` (`public` | `private`) are saved in the `forums` table and can be retrieved via the API after creation.  
3. **Editable configuration** – A subsequent `PUT /forums/:id` request updates any of the three fields and the changes are reflected in the DB.  
4. **Default permissions** – The newly‑assigned moderator role automatically includes the default permission set (`edit_posts`, `delete_posts`, `manage_members`).  
5. **Idempotent role assignment** – Re‑creating a forum with the same creator does not duplicate moderator role entries (no duplicate rows).  
6. **Security enforcement** – Only users with the `moderator` role for a given forum can access moderator‑only endpoints (e.g., `/forums/:id/moderate`).
