## One‑stop, production‑ready **docker‑compose.yml**  

Below is a single compose file that merges the strongest ideas from the two drafts:

| What we kept | Why it’s the best |
|--------------|-------------------|
| **Alpine‑based Postgres 15** and **Redis 7** images | Small footprint, multi‑arch, up‑to‑date security patches. |
| **`secrets`** for DB & Redis passwords (instead of hard‑coded values) | Prevents secrets from leaking into `docker inspect` or the compose file. |
| **Health‑checks** for every service | Docker will wait for a healthy DB/Redis before starting the app and will restart a failing container automatically. |
| **`depends_on` with `condition: service_healthy`** | Guarantees the app only starts after DB & Redis are reachable. |
| **Explicit bridge network** (`axentx-net`) | Isolates the stack from other containers on the host. |
| **Volumes for persistence** (`axentx‑db‑data`, `axentx‑cache‑data`, `axentx‑config`, `axentx‑data`) | Guarantees data survives container recreation. |
| **Port binding to localhost only** (`127.0.0.1:8080:8080`) | The API is not exposed to the public network unless you deliberately change it. |
| **Optional `build:` block** for the app | Lets you either pull a pre‑built image (`axentx/social-assembly:latest`) **or** build locally from a Dockerfile. |
| **Clear naming** (`axentx‑db`, `axentx‑cache`, `axentx‑app`) | Improves readability and avoids clashes with other projects. |

  # -------------------------------------------------
  # PostgreSQL – the relational store
  # -------------------------------------------------
  axentx-db:
    image: postgres:15-alpine               # latest stable, small
    container_name: axentx-db
    environment:
      POSTGRES_USER: axentx
      POSTGRES_DB: axentx_social
      POSTGRES_PASSWORD_FILE: /run/secrets/db_password
    volumes:
      - axentx-db-data:/var/lib/postgresql/data

      - axentx-net

#### Edge Cases:
1. **Missing Docker**
   - Run on fresh VM without Docker
   - Assert: Build fails with clear message: "Docker not found"

2. **Port Conflict**
   - Start another service on port 5432 or 6379
   - Run `docker compose up`
   - Assert: Exit with error indicating port conflict

3. **Disk Full**
   - Fill /var/lib/docker to 95%
   - Run `docker compose build`
   - Assert: Build fails with disk space error
