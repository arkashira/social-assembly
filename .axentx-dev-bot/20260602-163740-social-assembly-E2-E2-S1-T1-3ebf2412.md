## Final Design – Moderation Rules API

Below is a **single, production‑ready implementation** that pulls the strongest ideas from both candidates:

* **Full Mongoose schema** with automatic timestamps, versioning, and soft‑delete (`archived` flag).
* **RESTful routes** that cover create, read, update, delete (soft‑archive), and list.
* **Request validation** with Joi – guarantees required fields and meaningful errors.
* **Moderator guard middleware** – protects all rule‑management endpoints.
* **Clear error handling** – consistent JSON error payloads.
* **Front‑end friendly** – every rule returned contains the same shape the forum UI expects.
* **Version history** – each `save` increments `version`; old records are kept in the same collection.

> **Tip** – keep the code in the same folder structure used by the project (`/backend/...`).  
> All files below are ready to drop into place.

### 1. `backend/models/ModerationRule.js`

```js
// backend/models/ModerationRule.js
const mongoose = require('mongoose');

const ModerationRuleSchema = new mongoose.Schema(
  {
    title:      { type: String, required: true, trim: true },
    description:{ type: String, required: true, trim: true },
    severity:   { type: String, enum: ['low','medium','high'], default: 'medium' },
    archived:   { type: Boolean, default: false },
    version:    { type: Number, default: 1 },
  },
  { timestamps: true }          // createdAt / updatedAt
);

// Increment version on every update (not on create)
ModerationRuleSchema.pre('save', function (next) {
  if (!this.isNew) this.version += 1;
  next();
});

module.exports = mongoose.model('ModerationRule', ModerationRuleSchema);
```

* Moderators can edit existing rules, and the changes are reflected in the database and the forum display.
* Moderators can archive rules, and the archived rules are no longer displayed in the forum but are still stored in the database for versioning and auditing purposes.
* Rules are versioned, with each edit creating a new version, and timestamped, with the creation and last update timestamps stored.
* The API endpoints for creating, editing, and archiving rules are secured, requiring authentication and authorization as a moderator.
* The API returns appropriate error messages for invalid requests, such as attempting to create a rule without a title or description.
