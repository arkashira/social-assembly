- **AC3** – The creator (authenticated moderator) is automatically assigned the **moderator** role for the new forum with the default permission set (e.g., `can_edit = true`, `can_delete = true`).
- **AC4** – The endpoint validates input and returns **400 Bad Request** with a descriptive error object when any field fails validation (e.g., missing name, name too short, invalid privacy value).
- **AC5** – After creation, a **GET /api/forums/:id** request returns the forum data matching the persisted values and includes a `role: "moderator"` field for the creator.
- **AC6** – The created forum can be updated via **PUT /api/forums/:id** by the same moderator; changes are persisted and reflected in subsequent GET calls.
- **AC7** – All database operations are performed within a transaction; on any failure, no partial data is left (no forum row without a moderator assignment).
