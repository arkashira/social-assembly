+# Deployment Guide
+
+This guide walks you through a **custom self‑hosted deployment** of **social‑assembly**.  
+It covers three core steps that every deployment needs:
+
+1. **Database setup** – PostgreSQL (recommended) or SQLite for quick testing.  
+2. **Reverse‑proxy configuration** – Nginx **or** Traefik.  
+3. **TLS/SSL** – Let’s Encrypt (automatic) or manual certificates.
+
+The guide also provides ready‑to‑copy example configuration snippets and a short **troubleshooting** section.
+
+---  
+
+## 1️⃣ Database Setup
+
+### 1.1 PostgreSQL (production‑grade)
+
+```bash
+# Install PostgreSQL (Debian/Ubuntu example)
+sudo apt-get update && sudo apt-get install -y postgresql postgresql-contrib
+
+# Start the service
+sudo systemctl enable --now postgresql
+
+# Create a dedicated role and database
+sudo -u postgres psql <<SQL
+CREATE USER social_assembly WITH PASSWORD 'CHANGE_ME_STRONG';
+CREATE DATABASE social_assembly OWNER social_assembly;
+GRANT ALL PRIVILEGES ON DATABASE social_assembly TO social_assembly;
+SQL
+```
+
+> **Tip:** Keep the password in a secret manager (e.g., Docker secrets, Vault) and reference it via an environment variable `DATABASE_URL`.
+
+### 1.2 SQLite (quick‑start / dev)
+
+```bash
+# No extra packages required – the binary ships with SQLite support.
+# Create an empty file that the app can write to:
+touch /var/lib/social-assembly/social.db
+chmod 600 /var/lib/social-assembly/social.db
+```
+
+> **Note:** SQLite is **not** recommended for production because it does not handle concurrent writes well.
+
+### 1.3 Environment variables
+
+| Variable          | Example (Postgres)                                 | Example (SQLite)                     |
+|-------------------|----------------------------------------------------|--------------------------------------|
+| `DATABASE_URL`    | `postgresql://social_assembly:CHANGE_ME@localhost:5432/social_assembly` | `sqlite:///var/lib/social-assembly/social.db` |
+| `DATABASE_POOL`   | `10` (max connections)                             | N/A                                  |
+
+Pass these variables to the container (Docker) or systemd service file.
+
+---  
+
+## 2️⃣ Reverse Proxy & SSL
+
+Social‑assembly runs an HTTP server on **port 3000** by default. A reverse proxy terminates TLS and forwards traffic to the internal port.
+
+### 2.1 Nginx Example
+
+Create `/etc/nginx/sites-available/social-assembly.conf`:
+
+```nginx
+server {
+    listen 80;
+    server_name example.com www.example.com;
+
+    # Redirect HTTP → HTTPS
+    return 301 https://$host$request_uri;
+}
+
+server {
+    listen 443 ssl http2;
+    server_name example.com www.example.com;
+
+    # ----- TLS -----
+    ssl_certificate /etc/letsencrypt/live/example.com/fullchain.pem;
+    ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;

+
+    # ----- Proxy -----
+    location / {
+        proxy_pass http
