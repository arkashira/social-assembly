# Deployment Guide

This guide provides step-by-step instructions for deploying the Social Assembly platform in a self-hosted environment.

## Prerequisites

- Docker and Docker Compose installed
- A domain name pointing to your server
- Basic understanding of Linux command line and networking

## Database Setup

Social Assembly requires a PostgreSQL database for persistent storage.

### Using Docker (Recommended)

Create a `docker-compose.yml` file with the following content:

```yaml
version: '3.8'

services:
  db:
    image: postgres:15
    environment:
      POSTGRES_DB: social_assembly
      POSTGRES_USER: social_user
      POSTGRES_PASSWORD: your_secure_password
    volumes:
      - postgres_data:/var/lib/postgresql/data
    networks:
      - social_network

  app:
    image: social-assembly/app:latest
    depends_on:
      - db
    environment:
      DATABASE_URL: postgresql://social_user:your_secure_password@db:5432/social_assembly
      # Add other required environment variables here
    ports:
      - "3000:3000"
    networks:
      - social_network

volumes:
  postgres_data:

networks:
  social_network:
```

Run the services:

```bash
docker-compose up -d
```

### Manual Installation

Install PostgreSQL:

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install postgresql postgresql-contrib

# CentOS/RHEL
sudo yum install postgresql-server postgresql-contrib
```

Create database and user:

```sql
CREATE DATABASE social_assembly;
CREATE USER social_user WITH PASSWORD 'your_secure_password';
GRANT ALL PRIVILEGES ON DATABASE social_assembly TO social_user;
```

Update the `database.yml` file with the new database credentials.

## Reverse Proxy Configuration

### Nginx Configuration

Create a new Nginx server block at `/etc/nginx/sites-available/social-assembly`:

```nginx
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

Enable the site and restart Nginx:

```bash
ln -s /etc/nginx/sites-available/social-assembly /etc/nginx/sites-enabled/
sudo systemctl restart nginx
```

### Traefik Configuration

```yaml
http:
  routers:
    social-assembly:
      rule: "Host(`yourdomain.com`)"
      service: social-assembly

services:
  social-assembly:
    loadBalancer:
      servers:
        - url: "http://localhost:3000"
```

## SSL Configuration

Obtain an SSL certificate from a trusted certificate authority. Update the `nginx.conf` or `traefik.yml` file to include the SSL certificate and private key.

For Nginx:

```nginx
server {
    listen 443 ssl;
    server_name yourdomain.com;

    ssl_certificate /path/to/your/certificate.crt;
    ssl_certificate_key /path/to/your/private.key;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```
