## Final Deployment Guide – Social Assembly (Self‑Hosted)

> **TL;DR**  
> ```bash
> # 1. Install prerequisites
> sudo apt update && sudo apt install -y git python3-pip python3-venv postgresql postgresql-contrib nginx certbot
> # 2. Clone repo & install Python deps
> git clone https://github.com/axentx/social-assembly.git
> cd social-assembly
> pip install -r requirements.txt
> # 3. Create DB user & database
> sudo -u postgres psql -c "CREATE USER sa_user WITH PASSWORD 'sa_pass';"
> sudo -u postgres psql -c "CREATE DATABASE sa_db OWNER sa_user;"
> # 4. Apply migrations
> python manage.py migrate
> # 5. Start the app (or use systemd)
> python manage.py runserver 0.0.0.0:8000
> # 6. Configure reverse‑proxy (see below)
> ```

---

## 1. Prerequisites

| Component | Minimum version | Install command |
|-----------|-----------------|-----------------|
| Ubuntu | 22.04 LTS (or Debian 12) | `sudo apt update && sudo apt upgrade -y` |
| Python | 3.10+ | `sudo apt install -y python3 python3-venv python3-pip` |
| PostgreSQL | 15+ | `sudo apt install -y postgresql postgresql-contrib` |
| Nginx | 1.22+ | `sudo apt install -y nginx` |
| Certbot (Let’s Encrypt) | latest | `sudo apt install -y certbot python3-certbot-nginx` |
| Git | latest | `sudo apt install -y git` |

> **Tip:** Create a dedicated system user (e.g., `sa_user`) to run the application.

## 2. Database Setup

```bash
# Switch to postgres user
sudo -u postgres psql

# Create role
CREATE USER sa_user WITH PASSWORD 'sa_pass';

# Create database owned by that role
CREATE DATABASE sa_db OWNER sa_user;

# Optional tuning
ALTER ROLE sa_user SET client_encoding TO 'utf8';
ALTER ROLE sa_user SET default_transaction_isolation TO 'read committed';
ALTER ROLE sa_user SET timezone TO 'UTC';

\q
```

**Apply migrations**

```bash
cd /opt/axentx/social-assembly
python manage.py migrate
```

**Optional: Enable autovacuum**

Add to `/etc/postgresql/15/main/postgresql.conf`:

```conf
autovacuum = on
autovacuum_naptime = 1min
```

## 3. Reverse‑Proxy Configuration

### 3.1 Nginx

Create `/etc/nginx/sites-available/social_assembly`:

```nginx
server {
    listen 80;
    server_name your_domain.com www.your_domain.com;

    # Redirect HTTP to HTTPS
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name your_domain.com www.your_domain.com;

    ssl_certificate /etc/letsencrypt/live/your_domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your_domain.com/privkey.pem;
    include snippets/ssl-params.conf;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 3600;
    }
}
```

Enable and reload:

```bash
sudo ln -s /etc/nginx/sites-available/social_assembly /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 3.2 Traefik (optional)

Add to `traefik.toml`:

```toml
[http.routers]
  [http.routers.social_assembly]
    rule = "Host(`your_domain.com`)"
    service = "social_assembly_service"
    entryPoints = ["websecure"]
    tls.certResolver = "le"

[http.services]
  [http.services.social_assembly_service.loadBalancer]
    [[http.services.social_assembly_service.loadBalancer.servers]]
      url = "http://127.0.0.1:8000"
```

## 4. SSL Setup

### 4.1 Let’s Encrypt (certbot)

```bash
sudo certbot --nginx -d your_domain.com -d www.your_domain.com
```

> Certbot will automatically renew certificates. Verify renewal with:

```bash
sudo certbot renew --dry-run
```

### 4.2 Manual SSL (self‑signed)

```bash
sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /etc/ssl/private/sa.key \
  -out /etc/ssl/certs/sa.crt \
  -subj "/CN=your_domain.com"
```

Update Nginx config to point to these files.
